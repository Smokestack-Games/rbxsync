File Format

The rbxjson Format

RbxSync uses .rbxjson files to represent non-script instances like Parts, Models, Frames, and more. Each file contains the className, instance name, and all properties with their types and values.

Our interpreter understands every property type: Vector3, CFrame, Color3, UDim2, Enums, and more. Properties are stored with explicit types so nothing is lost in translation.

Coming Soon: Full LSP support with autocomplete for every class, property, and value based on the Roblox API documentation. Create instances with intelligent suggestions.

Supported property types: String, Int, Float, Bool, Vector2, Vector3, CFrame, Color3, Color3uint8, BrickColor, UDim, UDim2, Rect, NumberRange, Font, Enum, Content, and more.

Command Line

Powerful CLI

Full-featured command line interface with all the commands you need.

rbxsync init

Initialize a new RbxSync project with standard directory structure.

rbxsync serve

Start the sync server. Connects to Studio plugin for live sync.

rbxsync build

Build project to .rbxl or .rbxm file. Supports all property types.

rbxsync build --watch

Watch for file changes and auto-rebuild. Great for rapid iteration.

rbxsync build --plugin

Output directly to Studio's plugins folder. Auto-installs on build.

rbxsync build -f rbxmx

Build to XML format (.rbxlx, .rbxmx). Human-readable output.

rbxsync build-plugin

Build the Studio plugin from source. Use --install to auto-install.

rbxsync sourcemap

Generate sourcemap.json for Luau LSP integration.

rbxsync fmt-project

Format all .rbxjson files with consistent style. Use --check for CI.

rbxsync doc

Open RbxSync documentation in your browser.

rbxsync studio

Launch Roblox Studio with optional place file.

rbxsync extract

Extract game from connected Studio to local files.

Get Started

Installation Guide

Up and running in minutes with a few simple commands.

1

Build from Source

Clone the repository and build with Cargo. Requires Rust to be installed.

# Clone and build
git clone https://github.com/devmarissa/rbxsync
cd rbxsync
cargo build --release

# Add to PATH (optional)
cp target/release/rbxsync /usr/local/bin/
2

Install Studio Plugin

Build and install the Roblox Studio plugin.

rbxsync build-plugin --install

# Or manually: the plugin will be at
# ~/Documents/Roblox/Plugins/RbxSync.rbxm
3

Install VS Code Extension (Optional)

For the best experience, install the VS Code extension.

# Build the extension
cd rbxsync-vscode
npm install
npm run build

# Install the .vsix file:
# Option 1: Command line
code --install-extension rbxsync-*.vsix

# Option 2: VS Code UI
# Press Cmd+Shift+P (Mac) or Ctrl+Shift+P (Win)
# Type "Install from VSIX" and select the file
4

Initialize Project

Create a new project or use an existing directory.

rbxsync init --name MyGame
rbxsync serve
5

Connect Studio

Open Roblox Studio and restart it to load the plugin. Set the project path in the RbxSync widget and click Connect. You're ready to sync!
